# BCABuddy – Development Roadmap

This roadmap defines how BCABuddy will be built step by step.
The focus is on simplicity, free infrastructure, and real student needs.

---

## Phase 0 – Foundation (Completed)

✔ GitHub repository setup  
✔ Folder structure finalized  
✔ Project identity defined  
✔ Persona and tone locked  
✔ Clear mission and scope  

---

## Phase 1 – Public MVP (Current Focus)

Goal: Allow IGNOU BCA students to use BCABuddy freely without login.

Features:
- Public anonymous access
- Buddy intro screen
- Mode selection:
  - Assignment
  - PYQ
  - Notes
  - Viva
  - Lab
  - Summary
- Hinglish default responses
- English on request
- Cloud-based AI
- RAG using IGNOU PDFs
- No user data stored

Deployment:
- Free cloud hosting
- No paid services required

---

## Phase 2 – Save & Export (Login Introduced)

Goal: Add value-based login without forcing users.

Features:
- Login only when user wants to:
  - Save notes
  - Bookmark answers
  - Export content
- Authentication:
  - Email + Password
  - Google Login
- Export formats:
  - PDF
  - DOCX
  - TXT

Anonymous users remain fully supported.

---

## Phase 3 – Personalization

Goal: Improve experience for logged-in users.

Features:
- Saved history
- Bookmarks
- Preferred modes
- Language preference
- Subject focus
- Cleaner UI interactions

---

## Phase 4 – Academic Intelligence

Goal: Improve exam relevance and usefulness.

Features:
- PYQ pattern analysis
- Important topic suggestions
- Revision-friendly summaries
- Exam-oriented answer formatting
- Improved lab explanations

---

## Phase 5 – Expansion (Optional)

Goal: Grow only if stable and useful.

Possible expansions:
- Other IGNOU programs
- MCA support
- Additional universities
- Community-shared notes

Expansion will happen only if:
- Core BCA experience is stable
- Infrastructure remains free
- Quality is not compromised

---

## Guiding Principles

- Students first
- Free forever for core features
- No dark patterns
- No forced login
- No ads
- No payment pressure

---

## Final Note

BCABuddy is built slowly and thoughtfully.

If a feature adds stress, cost, or confusion —  
it does not belong in the product.
